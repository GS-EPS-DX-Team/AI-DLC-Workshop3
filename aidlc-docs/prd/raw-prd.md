# PRD — 말해 뭐해
 
> 버전: v0.1 (워크샵 실습용 초안 / 배포 후 폐기 예정)
> 작성일: 2026.04.22
 
---
 
## 1. 제품 개요
 
현업 담당자가 텍스트·음성·파일 등 다양한 형태로 제출한 요구사항을 AI가 주제 테마별로 자동 분류하고, 구조화된 유저스토리 형식으로 변환한다. PO(관리자)는 생성된 유저스토리 목록을 검토·선택하고, 선택된 항목의 목적과 AC(인수 조건)를 즉시 출력할 수 있다.
 
---
 
## 2. 배경 및 문제 정의
 
| 문제 | 설명 |
|------|------|
| 현업 의견 수집의 파편화 | 요구사항이 메일·메모·구두 보고 등 다양한 채널에 분산되어 누락이 발생한다 |
| 유저스토리 작성 병목 | PO가 현업 의견을 직접 정리해 유저스토리로 변환하는 데 많은 시간이 소요된다 |
| 형식 불일치 | 작성자마다 유저스토리 포맷이 달라 품질이 일정하지 않다 |
| 맥락 손실 | 구두로 전달된 현업 의견은 회의 후 빠르게 잊히거나 왜곡된다 |
 
---
 
## 3. 목표
 
- 현업 담당자가 형식에 구애받지 않고 요구사항을 자유롭게 입력할 수 있도록 한다
- AI가 입력된 내용을 주제별로 분류하고, 일관된 형식의 유저스토리 초안을 자동 생성한다
- PO가 초안을 검토·선택하여 즉시 사용 가능한 유저스토리 산출물을 출력할 수 있도록 한다
 
---
 
## 4. 사용자
 
| 역할 | 설명 | 주요 목적 |
|------|------|-----------|
| 현업 담당자 | 요구사항을 제출하는 실무자 (운전원, 정비원, 관리자 등) | 불편·개선 의견을 빠르게 전달 |
| PO (관리자) | 유저스토리를 검토하고 최종 산출물을 출력하는 담당자 | 현업 의견을 유저스토리로 정제·선택·출력 |
 
---
 
## 5. 기능 범위 (In Scope / Out of Scope)
 
**In Scope**
- 멀티모달 입력: 텍스트 직접 입력, 음성 녹음, 파일 업로드(문서·이미지 등)
- AI 기반 주제 테마 자동 분류
- 유저스토리 초안 자동 생성 (유저스토리 문장 + 목적 + AC)
- PO 검토 화면: 테마별 유저스토리 목록 조회 및 체크 선택
- 선택된 유저스토리의 목적·AC 출력 (화면 출력 및 파일 다운로드)
 
**Out of Scope**
- 생성된 유저스토리의 Jira·노션 등 외부 툴 자동 연동
- 현업 담당자의 유저스토리 직접 수정
- 유저스토리 이력 관리 및 버전 관리
